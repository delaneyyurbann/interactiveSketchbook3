Heuristic Reading Response

I think heuristic number one “Visibility of System Status” is most aligned with the reading from Nielson. Heuristic number one states “The design should always keep users informed about what is going on, through appropriate feedback within a reasonable amount of time”. Norman’s entire concept that he created is described through terms of affordances, signifiers, and feedback. The affordances are what the system is capable of performing, but without a signifier the user is unable to know what is able to be afforded, and without feedback the user is unable to see what has been afforded to know if the system is working properly and determine the state of the system. This first heuristic is essentially describing this process of feedback and signifiers of feedback in this affordance system that Norman has created.


Another heuristic that I could relate to Norman’s first chapter, is number 5: “Error Prevention”. This heuristic states: “Good error messages are important, but the best designs carefully prevent problems from occurring in the first place. Either eliminate error-prone conditions, or check for them and present users with a confirmation option before they commit to the action.” This heuristic directly relates to the human-centric design that Norman discusses in his book. Designers should design for people and take into account all the errors they could possibly make, a designer shouldn’t assume a user knows anything about a product and should therefore design considering all correcting all possible errors before the human is able to make them. This idea reiterates the idea that designers should be both engineers and psychologists because not only should we design good systems that perform well, but we should also understand the human mind to predict how a user will interact with the system.


I think my micro-project violates heuristic number 6 Recognition vs Recall. Recognition vs Recall states: “Minimize the user's memory load by making elements, actions, and options visible. The user should not have to remember information from one part of the interface to another. Information required to use the design (e.g. field labels or menu items) should be visible or easily retrievable when needed.” Once a fact is unlocked and you click the next one, the fact disappears, which relies on the memory of the user. If I were to fix this I would add a specific back button to return to the previous fact. Secondly, my micro-project violates heuristic number 3 User Control and Freedom. This heuristic states: “Users often perform actions by mistake. They need a clearly marked "emergency exit" to leave the unwanted action without having to go through an extended process.” My system violates this because there is no hard restart button allowing the user to go back to the beginning, once you’ve started the system you have to go all the way through it to reach the end then press the refresh button to restart the task. If I were to fix this I’d add a button saying “start over” that could be pressed at any point during the game. In terms of my sketchbook as a whole, the main navigation bar is only on my HTML, so if I wanted to make my sketchbook more user friendly I’d add my nav bar to all pages of the site.
