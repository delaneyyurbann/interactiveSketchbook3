Structural Reflection

This week I focused on locking in a clear structure for how the sketchbook actually works rather than just how it looks. The main decision I made was to separate everything into three consistent areas on desktop: a navigation column on the left, a large central space for the active sketch, and a smaller area for text. Instead of everything flowing in one long scroll, each part now has a defined role. On mobile, I translated that same system into a vertical layout with a dropdown menu replacing the sidebar so all the sketches are still accessible without taking up space. From my previous version, the biggest change was moving away from a linear layout where users had to scroll or click through things in sequence. Now the list of sketches is always available, which makes it easier to jump between them without losing context. I also simplified the interaction space into one main container. Before, there were multiple areas competing for attention, and it was not always clear where the actual interaction was supposed to happen. Now it is more focused, with one active sketch at a time. This directly addresses a problem I identified in my audit around unclear hierarchy and inefficient navigation. Earlier, the structure made it hard to understand where you were or how to move between sketches. By making navigation persistent and clearly separating it from the interaction space, the system feels more stable and easier to use. There are still some things that feel unresolved. The relationship between the interaction area and the text section is not fully figured out yet. I am not sure if the text should update with each sketch, stay static, or act more like a reflection space. Also, as I add more sketches, the navigation list could become too long, so I will probably need to think about grouping or organizing it differently. Finally, the “go back” button might not make sense anymore since the navigation is always visible, so I need to decide if it is necessary or just leftover from the old structure.
