Structural Reflection

This week, I focused on refining the structural system of my interactive sketchbook by clearly defining three mobile page types: the Home page, an individual Sketch page, and a Micro-Project page. My main goal was to create a more consistent layout system across these pages while prioritizing interaction as the central component of the experience. One of the primary structural decisions I made was to standardize the layout across all three screens. Each page now follows a similar order: a persistent header (H1), a menu/navigation entry point, a content or instruction area, a dominant interaction space, and action buttons. This consistency helps establish a predictable user flow and reinforces the idea of the sketchbook as a unified system rather than a collection of disconnected pages. Compared to my previous iteration, I reduced layout variation and removed competing structures that made the system feel fragmented. In earlier versions, different pages used different spatial arrangements, which made it unclear where users should focus their attention. In this iteration, I emphasized the interaction area by giving it the most visual weight and positioning it consistently across screens. I also introduced repeated interaction controls such as “start over” and “go back” to establish consistent interaction cues. This iteration addresses a key problem identified in my audit: a lack of clear content prioritization and inconsistent interaction flow. By restructuring the pages, I aimed to guide the user more clearly from instruction to interaction. However, system status remains underdeveloped. The current design does not yet clearly indicate which sketch or section the user is in, nor does it provide feedback after interactions occur. Additionally, while the navigation model has been simplified into a single “menu” entry point, it still needs to be more explicitly resolved as a mobile-first pattern. The hierarchy beyond the main heading also needs further clarification, as secondary and tertiary levels are not yet fully defined through labeling. Overall, this iteration establishes a more consistent structural foundation, but hierarchy clarity, system status, and navigation behavior remain areas for further refinement.
