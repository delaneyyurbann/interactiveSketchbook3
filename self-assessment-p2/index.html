Reflection

This system is basically designed to be as simple as possible. The idea is an “interactive sketchbook,” so instead of building something with a lot of depth or complicated flows, I focused on making it easy to move between different sketches quickly. It’s not really about completing a task, it's more about exploring. The main navigation is a dropdown menu that exists on every page. When you tap “menu,” it opens a list of all the sketches, and you can jump directly to any of them. There’s no multi-step navigation or deep hierarchy. Everything is one click away. I did this on purpose because I didn’t want users to feel like they had to search for things or remember where something was. It keeps the experience really lightweight. Even though it’s simple, there is still some structure. The first page acts as a kind of starting point, but after that, all the pages are treated the same. “Sketch 1,” “Micro-Project,” etc. all have the same layout and importance. There isn’t really a “main” page after the start. That was intentional. I wanted it to feel more like flipping through a sketchbook than moving through a strict system. I designed everything mobile-first, which is why the layout is very vertical and broken into clear sections. Each screen has a header, a main interaction area, and a smaller content section. There’s not a lot on the screen at once, and everything is sized for tapping. The dropdown menu also makes more sense on mobile because it saves space and keeps navigation consistent without cluttering the screen. Looking at Norman’s ideas, the system focuses a lot on making things obvious. The menu is clearly labeled, and it behaves how you expect, it opens, you pick something, and you go there. There aren’t hidden interactions or confusing states. The goal was to make it so you don’t have to think about how to use it. You just see it and understand it. With Nielsen’s heuristics, consistency is probably the biggest one. Every page looks and works the same way, so once you understand one, you understand all of them. There’s also a lot of user control, you can go anywhere from anywhere, and you’re not stuck in a flow. The “go back” and “start over” buttons help with that too. There’s not much room for error because the system itself is pretty simple. Shneiderman’s rules show up in how predictable everything is. Actions always do what you expect, and you can always undo or go back. Nothing feels permanent or risky. The idea is that users should feel comfortable clicking around without worrying about messing anything up. Keeping the layout and interactions consistent across screens also helps reinforce that. Overall, the system is intentionally minimal. I wasn’t trying to build something complex, I wanted something clear and easy to move through. The focus is on the content (the sketches), not the interface. The interface just supports that by staying out of the way.
